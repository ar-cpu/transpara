     node --version
     npm --version

   If you see "npm is not recognized":
   - Close all terminals
   - Open a NEW PowerShell window
   - Try again
   - If still fails, restart your computer

   See SETUP_NODEJS.md for detailed Node.js installation help!

3. Install Git for Windows:
   - Download: https://git-scm.com/download/win
   - Run installer with default settings

USEFUL DOCKER COMMANDS
===================================================================

View logs from all services:
  docker-compose logs -f

View logs from specific service:
  docker-compose logs -f backend
  docker-compose logs -f postgres
  docker-compose logs -f redis

Stop all containers:
  docker-compose down

Restart all containers:
  docker-compose restart

Restart specific service:
  docker-compose restart backend

Check container status:
  docker-compose ps

Stop and remove everything (including data):
  docker-compose down -v
  (WARNING: This deletes all user accounts and analysis history!)

Start containers again:
  docker-compose up -d

HOW AUTHENTICATION WORKS
===================================================================

1. USER REGISTERS:
   Frontend â†’ POST /api/v1/auth/register
   Backend â†’ Hash password with bcrypt
   Backend â†’ Save to PostgreSQL users table
   Backend â†’ Return success message

2. USER LOGS IN:
   Frontend â†’ POST /api/v1/auth/login (username + password)
   Backend â†’ Verify password hash
   Backend â†’ Generate JWT access token (expires 1 hour)
   Backend â†’ Generate JWT refresh token (expires 30 days)
   Backend â†’ Return tokens + user info
   Frontend â†’ Store tokens in localStorage
   Frontend â†’ Close login modal, show dashboard

3. USER MAKES API REQUEST:
   Frontend â†’ Add header: Authorization: Bearer <access_token>
   Backend â†’ Verify JWT signature
   Backend â†’ Check token expiration
   Backend â†’ Extract user ID from token
   Backend â†’ Process request
   Backend â†’ Return response

4. USER LOGS OUT:
   Frontend â†’ Remove tokens from localStorage
   Frontend â†’ Redirect to login modal

Problem: "docker-compose is not recognized"
Solution:
  - Make sure Docker Desktop is running (check system tray)
  - Restart terminal after installing Docker
  - Try: docker compose up -d (without hyphen)

Problem: "port is already allocated"
Solution:
  - Check what's using the port:
    netstat -ano | findstr :80
    netstat -ano | findstr :5432
  - Kill process: taskkill /PID <process_id> /F
  - Or stop IIS if running: iisreset /stop

Problem: Containers keep restarting
Solution:
  - Check logs: docker-compose logs -f backend
  - Check .env file exists in backend folder
  - Verify SECRET_KEY and DATABASE_URL are set
  - Restart Docker Desktop

Problem: "database does not exist"
Solution:
  - Stop containers: docker-compose down -v
  - Start again: docker-compose up -d
  - Wait 15 seconds
  - Run migrations: docker-compose exec backend flask db upgrade
